Administration commands

add_user

Inserts a new user with the given parameters.
add_user(name, email, institution, user key)

Parameters:

  • name (string) — name for the new user
  • email (string) — email address of the new user
  • institution (string) — institution of the new user
  • user key (string) — users token key

Responses:

  • ['okay', result] — result consists of
    • id (string) — id of the newly inserted user
  • ['error', error_message] — Error. Verify the error message.

delete

Remove data from DeepBlue.
delete(id, user key)

Parameters:

  • id (string) — Data id to be deleted.
  • user key (string) — users token key

Responses:

  • ['okay', result] — result consists of
    • id (string) — id of the deleted data
  • ['error', error_message] — Error. Verify the error message.

init_system

Initializes the system with the given user as administrator.
init_system(name, email, institution)

Parameters:

  • name (string) — name for the initial user
  • email (string) — email address of the initial user
  • institution (string) — institution of the initial user

Responses:

  • ['okay', result] — result consists of
    • admin_key (string) — administrator's authentication key
  • ['error', error_message] — Error. Verify the error message.

list_users

Lists all existing users.
list_users(user key)

  • user key (string) — users token key

  • ['okay', result] — result consists of

search

Search all data of all types for the given keyword. A minus (-) character in front of a keyword searches for data without the given keyword. The search can be restricted to the following data types are: annotations,bio_sources,epigenetic_marks,experiments,genomes,projects,samples,samples.fields,techniques,tilings
search(keyword, type, user key)

Parameters:

  • keyword (string) — keyword to search by
  • type ([string, ...]) — type of data to search for
  • user key (string) — users token key

Responses:

  • ['okay', result] — result consists of
    • results (array) — search results as [id, name, type]
  • ['error', error_message] — Error. Verify the error message.

Operating on the data regions

aggregate

Summarize the data regions content in range regions. Use the fields @AGG.MIN, @AGG.MAX, @AGG.MEDIAN, @AGG.MEAN, @AGG.VAR, @AGG.SD, @AGG.COUNT in the get_regions command format parameter for retrieving the computed values.
aggregate(data_id, ranges_id, field, user key)

Parameters:

  • data_id (string) — id of the query with the data
  • ranges_id (string) — id of the query with the regions range
  • field (string) — id of the aggregate regions range
  • user key (string) — users token key

Responses:

  • ['okay', result] — result consists of
    • regions (string) — BED formated regions
  • ['error', error_message] — Error. Verify the error message.
