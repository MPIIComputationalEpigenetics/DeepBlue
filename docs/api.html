Getting started

Please read our terms and conditions of use carefully before using DeepBlue Epigenomic Data Server.

In order to gain access to the DeepBlue API, you will first need to email us at deepblue@mpi-inf.mpg.de to obtain an user key. (Read more)

Once we issue you an user key, you will do all the following operations in your favorite language using XML-RPC. (Read more)

Throttle Limits

DeepBlue API user keys are free to do how many requests with per day.

The public users keys will be throttled as soon the server begins to receive many requests.

If you need more API calls per day, feel free to reach out to us at deepblue@mpi-inf.mpg.de.

Registering

Requests to the DeepBlue API must include the researcher name, researcher email, and the reseacher institution.

In order to obtain the users key, please email deepblue@mpi-inf.mpg.de with the required information. Remember that creating an user key is a manual process, so, it could take some days (specially on weekends and holidays).

We will anser you with a key. This key is your key to access all DeepBlue services. As a personal key, do not share it. Contact us as soon as possible if you lost or another person got access to this key.

After the registration, you can use DeepBlue Epigenomic Data Server.

Accessing DeepBlue

DeepBlue uses XML-RPC protocol for communication between user and the server. XML-RPC does not depend on the programming language. You can use your favorite programming language to use DeepBlue.

Accessing DeepBlue through:

  • Java — Apache XML
  • Python — xmlrpclib
  • R (bioconductor) — XMLRPC
  • PHP — XML-RPC
  • Others programming languages — google it

You can use the operation echo to test your key. Here is an exemple of accessing DeepBlue using python language: 

>>> import xmlrpclib
>>> address = "deepblue.mpi-inf.mpg.de/xmlrpc"
>>> port = 31415
>>> server = xmlrpclib.Server("http://"+address+":"+str(port), allow_none=True)
>>> server.echo("testuserkey123")
['okay', 'DeepBlue (0.3.0) says hi to Felipe Albrecht']

If your key is invalid, you will receive the following result: 

['error', 'Invalid user key.']

Administration commands

add_user

Inserts a new user with the given parameters.
add_user(name, email, institution, user key)

Parameters:

  • name (string) — name for the new user
  • email (string) — email address of the new user
  • institution (string) — institution of the new user
  • user key (string) — users token key

Responses:

  • ['okay', result] — result consists of
    • id (string) — id of the newly inserted user
